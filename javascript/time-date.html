Universal time

All Date objects are relative to Universal Time Coordinated (UTC), also known as the GMT time zone.

Generate today’s date

To generate today’s date as a date object, call the Date constructor.

const todaysDate = new Date();

And to convert it to plain text to display easily:

const dateString = Date(todaysDate).toString();

Setting sections of a Date

To set specific sections of a Date object, use these setter methods:

JavaScript Usage Notes
todaysDate.setFullYear() Sets the 4-digit year, according to local time
todaysDate.setMonth() Sets the month (0–11), according to local time 0 corresponds to January
todaysDate.setDate([1–31]) Sets the day of the month, according to local time If the date provided is outside range for that month (i.e. 0, or 30 for February), the month will update accordingly. 0 will set the date to the last day of the previous month, any values over the last day of the month will increment into the following month, and negative values will backtrack into the previous month.
todaysDate.setTime() Sets the exact time represented by milliseconds since Jan 1, 1970, 00:00:00.000 GMT (the EcmaScript Epoch), according to local time
todaysDate.setHours() Sets the hour (0–23), according to local time
todaysDate.setMinutes() Sets the milliseconds, according to local time
todaysDate.setSeconds() Sets the seconds (0–59), according to local time
todaysDate.setMilliseconds() Sets the milliseconds, according to local time
todaysDate.setUTCFullYear() Sets the 4-digit year, according to UTC time
todaysDate.setUTCMonth() Sets the month (0–11), according to UTC time 0 corresponds to January
todaysDate.setUTCDate() Sets the day of the month, according to UTC time If the date provided is outside range for that month (i.e. 0, or 30 for February), the month will update accordingly. 0 will set the date to the last day of the previous month, any values over the last day of the month will increment into the following month, and negative values will backtrack into the previous month.
todaysDate.setUTCHours() Sets the hours (0–23), according to UTC time
todaysDate.setUTCMinutes() Sets the minutes (0–59), according to UTC time
todaysDate.setUTCSeconds() Sets the seconds (0–59), according to UTC time
todaysDate.setUTCMilliseconds() Sets the milliseconds, according to UTC time

Outputting sections of a Date

All examples below will be using todaysDate to generate the output.

Note: If the time zone is not defined on a Date, it can lead to unexpected behavior. If the date output looks offset from what it should be, this is what’s happening.

Time

JavaScript Usage Notes Output
todaysDate.getTime() Returns the duration in milliseconds between the Date object and Jan 1, 1970, 00:00:00.000 GMT (the EcmaScript Epoch)
todaysDate.getTimezoneOffset() Returns the difference between the Date and UTC, in minutes This is not the same as the timezone offset; it merely gives you the difference in minutes
todaysDate.getHours() Returns the hour (0–23)
todaysDate.getMinutes() Returns the minutes (0–59)
todaysDate.getSeconds() Returns the seconds (0–60)
todaysDate.getMilliseconds() Returns the milliseconds

Date

All of these methods return values relative to the local time of the Date object.

JavaScript Usage Notes Output
todaysDate.getFullYear() Returns the 4-digit year
todaysDate.getMonth() Returns the month (0–11) 0 corresponds to January
todaysDate.getDay() Returns the day of the week (0–6) 0 corresponds to Sunday
todaysDate.getDate() Returns the day of the month

UTC

All of these methods return values relative to UTC, regardless of the local time of the Date object.

JavaScript Usage Notes Output
todaysDate.getUTCFullYear() Returns the 4-digit year
todaysDate.getUTCMonth() Returns the month (0–11) 0 corresponds to January
todaysDate.getUTCDate() Returns the day of the month (1–31)
todaysDate.getUTCDay() Returns the day of the week (0–6) 0 corresponds to Sunday
todaysDate.getUTCHours() Returns the hours (0–23)
todaysDate.getUTCMinutes() Returns the minutes (0–59)
todaysDate.getUTCSeconds() Returns the seconds (0–59)
todaysDate.getUTCMilliseconds() Returns the milliseconds

Manipulating a date with methods

JavaScript Usage Notes Output
todaysDate.toDateString() Returns a string in Ddd Mmm DD YYYY format Use this rather than .toString() since formats for .toString() may differ depending on platform and engine
todaysDate.toISOString Returns a string in simplified ISO format: YYYY-MM-DDTHH:MM:SS.mSSZ The time zone is always zero UTC offset, as denoted by Z on the end
todaysDate.toJSON() Returns a string representation of the Date object in YYYY-MM-DDTHH:MM:SS.mSSZ format
todaysDate.toLocaleDateString([locale], [options]) Returns a string with language-sensitive representation according to the user’s settings. Default output does not include time. The options object allows customization of the date string:
{ weekday: 'long', year: 'numeric', month: 'long', day: 'numeric' } etc.
todaysDate.toLocaleString([locale], [options] Returns a string with language-sensitive representation according to the user’s settings. Default output does include time. The options object allows customization of the date string:
{ weekday: 'long', year: 'numeric', month: 'long', day: 'numeric' } etc.
todaysDate.toLocaleTimeString([locale], [options]) Returns a string with language-sensitive representation of the time portion of the date The options object allows customization of the date string:
{ timeZone: 'UTC', timeZoneName: 'short' } etc.
todaysDate.toString() Returns a string
todaysDate.toTimeString() Returns a string of the time portion of the date in human-readable format
todaysDate.toUTCString() Returns a string using the UTC time zone
