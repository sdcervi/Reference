Declaring variables

ES6 JavaScript introduces two new declaration keywords: const and let.

Keyword Scope Usage Examples
var Global Allows declaration of mutable variables. New variables declared locally with the same name will overwrite global variables’s values. Avoid using var; use const or let instead.
const Block

Allows declaration of immutable (constant) block-scope variables. New variables declared locally with the same name will not overwrite global variables’s values. This is best for variables that don’t need to be updated or re-declared. const declarations must be initialized when declared.

Note that while objects declared with const cannot be updated, the properties and child objects within it can. The same goes for arrays; the array variable itself can’t be reassigned, but its child elements can.
  • Data structures
  • Dates and times
let Block Allows declaration of mutable block-scope variables. New variables declared locally with the same name will not overwrite global variables’s values. This is best for variables whose values need to change based on program flow. let declarations do not need to be initialized with a value when declared.
  • Counters and loop iterators
  • Results of logic evaluations (true/false with different values depending on outcome)

Assigning values and performing arithmetic

To assign or increment a variable, use the following commomly used operators. The full list of assignment operators can be found here.

Assignment operators

Operator Description Usage Example
= Assignment Sets a variable equal to a value x = 5;
+= Addition assignment Increments a variable by a value, or concatenates strings together x += 5;
x += 'string';
-= Subtraction assignment Subtracks a value from a variable’s value (only works on numbers) x -= 5;
*= Multiplication assignment Multiples a variable by a value (only works on numbers) x *= 5;
/= Division assignment Divides a variable by a value (only works on numbers) x /= 5;
%= Remainder assignment Assigns a variable the remainder of it divided by a value let a = 3;
a %= 2; // outputs 1
** Exponentiation assignment Multiples a variable by itself a specified number of times (sets its exponent) let a = 3;
a ** 2; // outputs 9

Arithmetic operators

Operator Description Usage Example
% Remainder Returns the integer remainder of dividing two values x = 12 % 5; // sets x to 2
++ Increment Adds 1 to the value of an operand x++;
-- Decrement Subtracts 1 from the value of an operand x--;
- Unary negation Returns the negation of an operand let x = 3; return (-x); // returns -3
+ Unary plus Attempts to convert the operand to a number if it isn’t one already +"3" // returns 3
+true // returns 1
** Exponentiation operator Calculates the base to the exponent power 2 ** 3 // returns 8

Setting default values

If a value input into a variable is null, you can set a default value to display instead.

let username = usernameInput || 'Default value';

This is known as short-circuit evaluation and allows for failsafes in case of invalid or null input.

Data types

Simple data types (number, boolean, null, undefined)

Numbers can be any number, either a whole number (integer) or with decimal points (float).

Booleans are a binary true or false, usually used as an indicator flag or an on/off switch.

Null variables have the intentional absence of a value; the variable exists but does not have a valid value. Example: let x = null;

Undefined variables have not yet been declared, or have not had a value defined. Example: let x;

Strings

Strings are how plain text or other simple non-numerical values are stored. For example, 'foobar' is a string instead of a number, array, etc.

Enclose strings in 'single quotes', "double quotes", or `backticks`; it doesn’t matter what method as long as it’s consistent. Single quotes and backticks are best when generating HTML code via JavaScript, to avoid having to escape double quotes. Backticks are used for template literals; more on this in a bit.

'This is a string.'
"This is also a string."
`And this is another string.`

To escape a character in a string and prevent it from terminating the string early, use a \ before the character.

'Use a backslash to make sure that any use of the same character doesn\'t break your string.'

Strings can be concatenated (added) with the + operator. When constructing strings of HTML code containing variables, however, this can get messy and lead to mistakes; it’s best to use template literals instead. Enclose variables and operations in ${curly braces} with a dollar sign preceding them.

`Use template literals to add in ${variables} directly to a string.`;

Arrays

Arrays are best used to hold sets of data that need to remain in a specific order. Their data can be accessed by numerical index, starting at 0.

To get the size of an array, use arrayName.length.

Methods

Arrays come with default methods attached to them, to perform basic operations. Use these methods with arrayName.method().

Method Usage
.push([data]) Adds data to the end of an array
.pop() Removes and returns the last element of an array
.unshift([data]) Adds data to the beginning of an array
.shift() Removes and returns the first element of an array
.slice([start], [end]) Copies the indicated range of an array into a new array; end is not included and the operation proceeds only up to end - 1. The original array is not modified.
.isArray([variable]) Returns true if the variable contains an array, false otherwise
.every([function]) Returns true if every element in the array satisfies the function
.filter([function]) Returns a new array containing all elements that satisfy the filtering function
.findIndex([function]) Returns the index of the first element where the function returns true. This is different from .indexOf([input]) because it allows you to check a logical condition. .indexOf() only returns the first element in the array whose contents match the input.
.forEach([function]) Executes a function for each element in the array; more on forEach() loops here
.join() Joins all array elements together into one string
.map([function]) Executes a function for each element in the array, and returns a new array of the results
.reduce((previous, current) => [function], initial)

Executes a user-supplied callback reducer function on each element of the array, passing in the return value from the previous execution. An initialValue may be supplied to use as the ‘previous value’ for the first iteration. A common use case is summing up all the values in an array, or concatenating an array into a string.

This method is not inherently mutating; it doesn’t change the array, but code inside the callback function might.
.reverse() Reverses the elements of an array in place; first becomes last, and vice versa
.some([function]) Returns true if at least one element in the array satisfies the function
.sort()

Sorts the elements of an array in place by converting all elements to strings and comparing their values. The original array is modified. The default sort order is ascending; to specify a different sort order, include a function as a parameter. Note that since numbers are converted to strings, with the default sort method, 80 will be placed before 9. undefined elements will be sorted to the end of the array. More information on the .sort() method

When comparing two elements, the return value governs how those elements are sorted relative to each other.

  • > 0: sorts the second element before the first
  • < 0: sorts the first element before the second
  • === 0: keeps the original order of the first and second element

Example, sorting ascending:

function sortArray (inputArray) {
     return inputArray.sort((input1, input2) => {
          if (input1 < input2) {
               return -1;
          } else if (input1 > input2) {
               return 1;
          }
          return 0;
     });
}

Example, sorting descending:

function sortArray (inputArray) {
     return inputArray.sort((input1, input2) => {
          if (input1 < input2) {
               return 1;
          } else if (input1 > input2) {
               return -1;
          }
          return 0;
     });
}

To easily sort numbers, you can use numbersArray.sort((input1, input2) => input1 - input2); for ascending, and numbersArray.sort((input1, input2) => input2 - input1); for descending.

Objects

Objects are collections of related data organized into key-value pairs. They are similar to arrays, except values can be accessed by name using dot notation: object.key1, and their order within the object may change without warning. For keys with spaces, variable names as keys, or key names that are enclosed with quotes, use bracket notation: object['key 2']. Access keys of child objects using object.childname.keyname.

const objectName = {
     key1: value1,
     'key 2': 'value 2'
};

Object properties can also be accessed with bracket notation: object[property], and this is most useful when iterating through an object, especially when using a variable as the iterator. For example:

for (const iterator in objectName) {
     let property = objectName[iterator];
     [code to execute for each iteration]
}

To remove a key or child object from an object, use delete objectName.key.

Object methods

Methods can be added to an object just like any other key-value pair; they can then be used with object.method([parameters]) syntax.

const objectName = {
     methodName () {
          [code to execute]
     },
     ...
};

Built-in methods

Object.keys(objectName) returns an array of all top-level keys in the object, converted to strings.

Object.entries(objectName) returns an array of all key-value pairs in the object as arrays.

For more built-in methods, read the MDN Object documentation.

Getters and setters

Getters and setters are two special types of methods that allow you to retrieve and set data within an object without exposing the object’s properties. For example:

const menu = {
     _courses: {
          appetizers: [],
          mains: [],
          desserts: [],
     },

     get appetizers () {
          return this._courses.appetizers;
     },
     set appetizers (appetizer) {
          this._courses.appetizers = appetizer;
     }
}

While almost all problems that can be solved with getters and setters can also be solved by directly accessing the data, the best use case justification is when you need to perform operations on the data before returning or setting it.

Note: getters and setters can’t share the same name with any other object properties. One workaround is to add an underscore before the property name. Getters and setters can, however, share names with each other; if you want to have a getter and setter for the same property, they can be named the same.

Possible use cases:

  • Capitalizing a string for output
  • Retrieving a user’s full name by concatenating their first and last names
  • Verifying data is of the correct type and not falsy/null before saving to the object’s property

When passing parameters to a setter method, use the same syntax you would to assign a variable:

objectName.setterName = value;

this

When using a method, getter, or setter, if you try to access an object’s property from that function, it won’t be able to because that’s outside of the method’s scope, unless you use this. USe the syntax this.propertyName to allow methods to access the object’s properties.

this references the calling object, the object that contains the method that is being called.

Arrow functions inherentely bind an already defined this that is not the calling object when used for an object method. This makes them not well suited to being object methods that need to access object properties. TL;DR: don’t use arrow functions for object methods.

Privacy

JavaScript does not have any built-in ability to make object properties private (unable to be changed or mutated) or public. However, you can notate that properties are supposed to be treated as private by adding an underscore to the beginning of the property name:

const objectName = {
     _key1: value1,
     'key 2': 'value 2'
};

This is most applicable when needing to retain data types for performing operations on data. For example, if you have a number, you may want to add or subtract a value. Indicate the property is private to remind yourself not to access or change that value. The best way to work with this is to set the property to private and use a setter to verify the value is of the correct data type before saving the new value.

Factory functions

When you need to create several objects with the same keys but different values, a factory function is the best way to do that. They accept the values as parameters, create the object, and return the complete object.

const factoryFunctionName = ([parameters]) => {
     return {
          param1: param1,
          param2: param2,
          method1 () {
               [code to execute on param3]
          }
     }
}

const newObject = factoryFunctionName ([parameters]);

This can also be destructured and shortened by using property-value shorthand:

const factoryFunctionName = ([parameters]) => {
     return {
          param1,
          param2,
          method1 () {
               [code to execute on param3]
          }
     }
}

const newObject = factoryFunctionName ([parameters]);

Destructured assignment

We can use a shorthand to extract values from an object:

const value = objectName.value
const { value } = objectName;

To grab multiple values to store in their own variables, use the following syntax:

const { value1, value2, value3 } = objectName;

We can also use destructured assignment to grab child properties:

const { childValue } = objectName.childProperty;

Nested child objects can also be accessed and destructured:

const { value1, value2, childObject: {childValue1, childValue2} } = objectName;

Note: be careful with destructured variable names. They must exactly match the properties they’s referencing, unless you specify a new variable name:

const { value1: newVar1, value2: newVar2, value3: newVar3 } = objectName;
