Adding JavaScript to an HTML page

Inline scripting

Javascript can be added inline, just like an HTML element. Use this for simple, quick operations not requiring a lot of logic or render time.

<p id="output"></p>
<script>document.getElementById('output').innerHTML = new Date();</script>

This is generally not considered best practice; best practice is to put all JavaScript into a separate file and link to it.

External file

Keeping JavaScript in an external file allows for separation of concerns, a development principle that separates a program or website into sections that each address a specific ‘concern’ of the project. Generally this is implemented with the HTML file only containing HTML markup, all of the styling being in a separate CSS file, and all of the Javascript being in its own file or files.

Add a JavaScript file to a page using <script src="path/to/file.js"></script>.

Loading JavaScript

HTML loads sequentially, meaning that it executes each line of code in the order it appears in the page. This leads to a few problems when integrating JavaScript, since this can negatively impact user experience.

Best practice is to place any <script> integrations at the bottom of the document, before the </body> tag. This enables the entire page to render before the scripts load, and prevents scripts from delaying page render.

defer and async

Using the defer attribute tells the browser to wait until the page is fully parsed and loaded before executing the script. Use this when a script requires interaction with the DOM to make sure the DOM is fully loaded before the script executes.

Using the async attribute will load the script asynchronously and still load the rest of the page while the script downloads. However, unlike defer delaying code execution until after the page is fully rendered, async delays code execution until the script itself is fully downloaded. This is best for scripts that have no dependency on other scripts, and have no other scripts dependent on them; it doesn’t matter exactly when the scripts execute. When done right, this improves page performance by speeding up load times.

Altering CSS

To alter the CSS of a particular element, access the style property. Syntax is a bit different; any hyphenated CSS styles are written in camelCase instead.

document.getElementById('divID').style.marginTop = '1em';

The Document Object Model

The Document Object Model (DOM) is a hierarchical tree representing the entire structure of an HTML page; this allows developers to traverse the tree using JavaScript.

Each element is represented as a node, and functions as a child object of its parent objects. Within the object are all the properties that JavaScript can access and alter; for example, the CSS classes, an img’s src attribute, or the element’s id.

To access the DOM, start with the document object. Here is a complete list of all the properties and methods included on the document object.

Modifying the DOM

When generating HTML to add into a webpage, use the following properties and methods. HTML can be generated as plain strings, using either 'single quotes' or "double quotes", or as `template literals with ${variables}`. Store these strings in a variable to easily add to the DOM, or add directly for simpler operations. Make sure to properly escape any special characters including quotes, to prevent the HTML and/or JavaScript from breaking.

DOM Events

Javascript events occur when a user interacts with the webpage. For example, a user might click on an element, scroll to a specific point, or upload a file via a dialog box. By assigning event listeners to detect these interactions, we can make pages respond to them.

Using .on[event] allows for assignment of one function, but .addEventListener(...) allows for chaining of functions. Best practice is to create a named function, then reference via name.

Using the document element for the event listener will allow for globally-fired events if needed (e.g. for keystrokes).

let eventTarget = document.getElementById('targetElement');

eventTarget.addEventListener('click', function() {
     [code to execute]
});

or

let element = document.querySelector('button');

element.onclick = function() {
     element.style.backgroundColor = 'blue';
};

The event object

JavaScript stores events as Event objects with their related data and functionalities as properties and methods. When an event is triggered, the event object can be passed as an argument to the event handler function. Access these properties with eventVariable.propertyName.

Property Usage
target References the element that the event is registered to
timeStamp Returns the time in milliseconds from the beginning of the time origin until the event was created. In most webpages, this is the time from when the window was loaded to the time the event was fired.
type Returns a string containing the event’s type
