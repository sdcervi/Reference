Kinds of errors

There are a few different kinds of errors you can experience, working with HTML, CSS, and JavaScript.

Error Usage
ReferenceError A variable or function can’t be found, or referenced. This halts execution of the code.
TypeError A value is not a valid type; for example, attempting to reassign a const variable. This halts execution of the code.

Constructing errors

Sometimes the built-in errors aren’t enough for a particular need. In this case, we can construct our own Error objects. These can be used as part of the program flow, for example in if...else blocks to display an error if a validation fails.

console.log(Error('Your password is too weak.'));

This by itself just creates a new error; it doesn’t stop the program from executing. To do that, use the throw keyword.

throw Error('Something wrong happened');

try...catch

Use try...catch blocks to anticipate and handle errors if they occur. try contains the code that may throw an error, and catch contains the code to handle that error if it occurs. This allows for better error handling, and for the code to continue executing after an error is thrown. This is especially useful if testing a value against a switch statement, where the error may change depending on the result, or when we rely on data from external sources that may or may not validate correctly.

Oftentimes catch will contain a console.log statement, but can also have logic to revert a value to defaults for example.

try {
     [code to attempt that may throw error]
} catch (errorMessage) {
     [code to handle that error]
}
