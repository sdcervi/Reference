Logical operators and comparison operators

Use logical and comparison operators to compare data and perform basic operations on it.

Logical operators

Operator Description Usage Example
&& and Multiple comparison statements must all be true for the whole to be true if ((x < 10) && (y > 5)) { [code to execute if both evaluate to true] }
|| or Any one of multiple comparison statements must evaluate to true for the whole to be true if ((x < 10) || (y > 5)) { [code to execute if either evaluates to true] }
! not Negates a statement to switch between true and false if (!(x == y)) { [code to execute if (x == y) evaluates to false] }

If statements and ternary operators

If statements

These are used to perform evaluations on data; for example, whether a number is greater than or less than another number.

if ([variable] [operator] [comparator]) {
     [code to execute if true]
}

Example:

if (inputNumber < 10) {
     alert ('The input number is less than 10');
}

Use else to specify what to do, should the if statement evaluate to false.

if (inputNumber < 10) {
     alert ('The input number is less than 10');
} else {
     alert ('The input number is not less than 10');
}

To stack multiple if statements on the same data, use else if. Make sure to follow them with an else block so that you can handle cases when all comparisons evaluate to false.

if (inputNumber < 10) {
     alert ('The input number is less than 10');
} else if (inputNumber > 10) {
     alert ('The input number is greater than 10');
} else {
     alert ('The input number is neither less than nor greater than 10, so it must be exactly 10');
}

Ternary operators

These can be used as a shorthand for if-else comparisons where you have only two cases: true or false. This is easiest to use when checking if a variable’s value is not null or false.

Ternary operators can decrease the number of lines of code needed to execute certain comparisons, but they can be less intuitive for less experienced developers to understand. This is mostly a problem when it comes to the maintainability of code, especially when working over long time frames or with multiple developers.

let testVariable = [value];
testVariable ? [code to execute if true] : [code to execute if false];

To use a ternary operator to check against a non-boolean value (i.e. a string, number, etc.), use this syntax:

let testVariable = [value];
testVariable [comparator] [comparison value] ? [code to execute if true] : [code to execute if false];

Example:

let username = 'John Doe';
username === 'John Doe' ? alert ('User is John Doe') : alert ('Error, incorrect user');

Here’s another example, using it in a function call with a return statement:

function getFee(isMember) {
     return (isMember ? '$2.00' : '$10.00');
}

console.log(getFee(true)); // expected output: "$2.00"
console.log(getFee(false)); // expected output: "$10.00"
console.log(getFee(null)); // expected output: "$10.00"

Switch statements

Switch statements are used when checking data against multiple possible values. This can help simplify nested if else statements into a more logical path.

Define a case for each option you want to execute unique code for, and use default to handle any cases that don’t trigger the other cases.

switch ([variable]) {
     case [first option] :
          [code to execute]
          break;
     case [second option] :
          [code to execute]
          break;
     case [third option] :
          [code to execute]
          break;
     case [... option] :
          [code to execute]
          break;
     default :
          [code to execute]
}

The break; statement prevents the switch from executing all cases. Best practices are to write the default case as the last in the set; as such, a break; statement isn’t required. If placing it elsewhere in the set, though, it should have one.

For loops

For loops are used to iterate through a set of data such as an array or object, and perform operations on each subset of data.

Using a counter

The most basic syntax uses a counter and increments it until it reaches a specific value. The below code will start the counter at 0 and increment it by 1 each iteration through the loop. When the counter reaches the max value, the loop will exit and stop without completing another iteration.

Starting the counter at 0 is recommended, since it will allow you to sequentially access arrays starting at their first index (arrayName[counter]). Note that this will require you to know the array’s length, or use arrayName.length for the max value.

Don’t forget the let keyword to allow the counter to be incremented.

for (let counter = 0; counter < [max value]; counter++) {
     [code to execute for each iteration]
}

Iterating through objects

To iterate through an object, for example to access all key-value pairs including child objects, you use similar syntax. It will execute until there are no more child objects within the parent object to operate on, then exit. Access the key within the object using bracket notation: object[key].

for (const child in objectName) {
     let childObject = objectName[child];
     [code to execute for each iteration]
}

Iterating through arrays and strings

To iterate through an array or string, you can use a for ... of loop, similar to the for ... in loop for objects.

for ([variable] of [array or string]) {
     [code to execute for each iteration]
}

forEach

For arrays, you can also use the .forEach(...) method to easily operate on each element without having to know the array size. This normally takes an argument of a callback function, either as a parameter or an anonymous function.

const fruits = ['apple', 'orange', 'banana'];
fruits.forEach(fruit => {
     [code to execute]
});

While loops and do...while

While loops

While loops iterate repeatedly while a specified condition evaluates to true. Make sure to define the initial value of whatever is getting evaluated outside of the loop.

while ([condition]) {
     ...
}

For example, the following code will loop through the while loop until counter is no longer less than 20.

let counter = 5;
while (counter < 20) {
     ...
     counter++;
}

The biggest thing to be aware of with while loops is to make sure the evaluated condition will eventually evaluate to false, otherwise you could get stuck in a forever loop.

Do...while

These are a form of while loops that execute the contained tasks at least once, no matter the condition, then repeats them while the specified condition evaluates to true. This is best if you want to make sure the code runs once no matter what.

let counter = 5;
do {
     ...
     counter++;
} while (counter < 20);
