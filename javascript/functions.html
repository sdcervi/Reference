Basics

A function is a reusable block of code that performs the same tasks every time it is called by another block of code. Functions can also accept parameters as inputs to perform tasks on; these are treated like pre-declared variables within the function.

Functions are hoisted, meaning that they can be accessed before they are defined. However, relying on this is not considered best practice and can sometimes lead to frustrating and unexpected bugs.

Functions also often have return statements that send a final output back to the code block that called them.

// Declare and define a function to add two numbers together, accepted as parameters
function addNumbers (num1, num2) {
     return num1 + num2;
}

// Call the function and send it two parameters
addNumbers (3, 5); // Expected output: 8

Default parameters

If there’s a chance that a function may not get parameters passed to it, or that the parameters may have an invalid value, you can set default values as fallbacks.

function addNumbers (num1 = 3, num2 = 5) {
     return num1 + num2;
}

Function expressions

To set a variable equal to the return value of a function without using an extra line of code, use the following syntax:

const variableName = function ([parameters]) {
     ...
};

A function without a name is an anonymous function; these are almost always part of function expressions. It’s best practice to use const to declare function expressions.

Arrow functions

Arrow functions allow you a simplified shorthand for writing function expressions:

const variableName = ([parameters]) => {
     ...
};

Arrow functions without parameters still need to have empty parentheses:

const variableName = () => {
     ...
};

Arrow functions with only one line of code to execute can also be condensed into a single line, without using curly braces:

const variableName = ([parameters]) => [code];

Passing parameters to functions via onclick="..."

When passing a parameter to a function via an onclick event, the parameters need to be wrapped in quotes, otherwise they will be treated like variables or other keywords and may trigger an undefined or null error.

<button onclick="myFunction('abc', '5')">Click to call function</button>

If doing this via JavaScript-generated HTML (for example, a delete button on each block of data generated when iterating through a data structure), use template literals to avoid breaking the JavaScript or the HTML.
