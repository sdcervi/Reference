About regular expressions

Regular expressions define patterns of characters that can be used to modify, validate, or perform other operations on strings. These patterns can include sequences of specific characters, uppercase or lowercase, numbers, punctuation, or other symbols.

Quantifiers

To specify how many times a specific character or match can occur in a pattern, use the table below.

Greedy quantifiers match as much as possible, every time; lazy quantifiers only proceed to the first match.

Operand Usage Example Example match
+ The preceding element match occurs one or more times (greedy) Version \w-w+ Version A-b1_1
+? The preceding element match occurs one or more times (lazy) \d+? 1 in 12345
{n} The preceding element match occurs exactly n times \D{3} ABC
{m,n} The preceding element match occurs at least m but no more than n times (greedy) \d{2,4} 156
{m,n}? The preceding element match occurs at least m but no more than n times (lazy) \d{2,4} ab of abcd
{n,} The preceding element match occurs at least n times \w{3,} regex_tutorial
* The preceding element match occurs zero or more times (greedy) A*B*C AAACC
* The preceding element match occurs zero or more times (lazy) A*? empty in AAA (because it matches at zero)
? The preceding element match occurs once or no times plurals? plural

Logic

To perform basic logic on a pattern, use the following table.

Operand Usage Example Example match
| Alternation/OR operand; must match either of the specified sub-patterns AA|BB BB
( ... ) Capturing group; groups a sub-pattern, also called a block or sub-expression. Captures the group for later re-use with a group index number. A(nt|pple) Apple (captures “pple”)
\1 Contents of capturing group 1, usually to repeat later in an expression. Captures the group for later re-use with a group index number. r(\w)g\1x regex
\n Contents of capturing group n, usually to repeat later in an expression; requires at least n capturing groups. n is a digit from 1-9. (\d\d)\+(\d\d)=\2\+\1 12+65=65+12
(?: ... ) Non-capturing group; matches a sub-pattern without saving it for later re-use. Same basic functionality as a capture group other than that, just without the storage overhead of saving the group for reuse. A(?:nt|pple) Apple

Anchors and boundaries

Operand Usage Example Example match
^ Start of string, or start of line depending on multiline mode (but when [^...] inside brackets, it means “not”) ^abc.* abc (at start of line)
$ End of string, or end of line depending on multiline mode .*? the end$ this is the end
\b Word boundary: position where one side only is a letter, digit, or underscore Bob.*\bcat\b Bob pet the cat
\B Not a word boundary c.*\Bcat\B.* copycats

Regular expression Usage Example matches
.* Any character, any number of times, except for a line break This is a regular expression.

/^[ \s]+/ Whitespace at start of line; can be used to remove leading whitespace in form entries during validation      This starts with a tab.

/^#[a-fA-F0-9]{6}$/ Hex color value; can contain exactly 6 characters from the specified ranges of a-f, A-F, and 0-9. Hex color codes are not case sensitive, but this regular expression allows for matching both uppercase and lowercase letters for better compatibility. #af2126
