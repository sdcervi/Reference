localStorage basics

When looking for a way to allow data to persist between browser sessions, a common solution is to set up user accounts and server-based data storage to store and retrieve user data. This isn’t always the easiest to implement, though, and is also unnecessary in some situations. localStorage is a mechanism that allows a browser to store key-value pairs. It functions in a similar way to browser cookies.

Use cases

Like cookies, localStorage can be used to retain user data, settings, and other information between browser sessions. This information is best limited to data that would not cause harm if revealed to a third party in any way.

Some possible uses include:

  • Cookie policy acceptance flag, set to true when a user has accepted the cookie policy so that the cookie banner doesn’t show up again
  • Non-sensitive user settings specific to each device, such as a dark mode or light mode interface preference

Limitations

localStorage has a few limitations to be aware of:

  • The stored data is specific to the browser and device; data can’t be synced between devices without other solutions added to it.
  • Users may lose their stored data if they clear their cache or browsing data.
  • Very few users have the technical knowledge to extract their localStorage data from a browser; this is increasingly difficult on mobile and tablet devices.
  • localStorage can be exploited by hackers and is not considered secure enough to store sensitive data
  • localStorage is limited to 5MB total storage per website per browser
  • localStorage operations are synchronous and execute sequentially

Accessing, using, and setting localStorage values

When using localStorage, there are five methods that can be used:

  • setItem(): Add key and value to localStorage, as parameters
  • getItem(): This is how you get items from localStorage; it accepts a single parameter of the key name to retrieve
  • removeItem(): Remove an item by key from localStorage; it accepts a single parameter of the key name to remove
  • clear(): Clear all localStorage records for your website
  • key(): Passed a number to retrieve the key of a localStorage; most useful when looping through all keys in localStorage

Use these methods with window.localStorage.methodName([parameters]);. Note that localStorage can only contain strings, numbers, and boolean values, so any JSON data, objects, or arrays must be stringified when saving and parsed when reading.

window.localStorage.setItem('userData', JSON.stringify(userData));

It’s considered good practice to verify a localStorage key exists before trying to access it; this way, if the user is new to your site, you can set up a failsafe that initializes a localStorage key if they don’t have anything previously saved there.

if (window.localStorage.getItem('keyName')) {
     keyInput = window.localStorage.getItem('keyName');
} else {
     window.localStorage.setItem('keyName', '');
     keyInput = '';
}
