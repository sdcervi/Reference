Main principles

Properly executed animations should be like effective waitstaff—no one should notice them unless you’re actively trying to draw their attention in that moment.

Keep animations consistent throughout an interface, and make sure every animated entrance has a consistent exist.

Avoid any rapidly flashing animation, especially large areas of red. Current W3C guidelines advise elements flash no more than twice per second. Try to signal oncoming animations with a subtle motion first (i.e. a sidebar bumping slightly before it fully expands upon click).

Types of animations

Term Action Usage examples
Transitions Old imformation moves out of view and new information takes its place; these move the user through the space and reinforce where they were, where they are, and where they’re going next Main content sliders, search overlays
Supplements Shows ancillary information to the user without removing them from the current information state Dropdowns, alerts, tooltips, sidebars sliding in and out
Feedback Provide users with direct feedback about their interactions Button pushed states, swiping gesture, loading icons
Demonstrations Show how something works, put information into perspective, provide context, bring new information into the space “Processing” pages that tell the story of your data as you wait for it to load, illustrative animations to explain how a product functions
Decorations Flavors existing information without adding to or modifying it Loading screens that aren’t demonstrative, imagery to reinforce a brand

When is animation needed?

  1. Find out which animation type it is. For example, if it’s both a transition and a demonstration, then it adds even more valuable information and is likely worth implementing.
  2. Ask these questions:
    Does it show the user where new information came from and where old information went?
    Does it indicate progress?
    Does it intuitively move the user through the information space?
    Does it reinforce physics or branding?
    Does it explain something faster and more intuitively than words or a video could?
    The more of these boxes are checked, the more likely it is to be a net gain for your users.
  3. Animations can then be sorted into four categories:
    Low hanging fruit are the big wins at low costs. Prioiritize these first.
    Long-term investments will help people a lot but require more resources to implement. Run prototypes by users to decide whether to implement.
    Pet projects are great time-fillers for slow days but should be lower priority.
    Hopeless causes are both unneccessary and difficult to implement. You will never have the time or justification to work on these.

Don’t add an animation unless it is both necessary and useful; but if it’s both necessary and useful, make it beautiful, too.

Terminology

Term Definition Usage examples
easing The rate at which visual changes occur
  • Acceleration is good for system-initiated animations, like newsletter pop-ups. Use this to avoid startling users.
  • Deceleration starts quickly and gives the UI a snappy, responsive feeling. Use this for user-initiated interactionws like button clicks and page transitions
  • Speeding up then slowing down is best when moving elements toward each other.
  • Linear transitions work best for fades and color changes
  • Bounces can draw attention to elements, or add an air of “fun” to the UI.
duration How long an animation lasts
  • 100ms or less feels instantaneous
  • 215ms is the average response time to changes
  • 200-300ms is the currently accepted sweet spot
  • Up to 1s still feels connected
  • Over 10s disconnects users

Animations in the center of the cone of vision do better with short durations, and animations on the edge/in peripheral vision do better with longer durations.

Color and opacity changes do better with short durations, but movement often needs slightly longer durations.
properties What changes
  • Color & opacity convey fades
  • Location coveys motion
  • Scale and shape convey deformation and transformation

CSS timing functions

CSS has several built-in timing functions (see below), but they can also be custom defined using cubic Bézier curves. They follow the format cubic-bezier(w, x, y, z) and can be used to create single and double bounces. Use the tools at cubic-bezier.com and easings.net to help decide which easing function to use.

Function Bezier curve Description
ease (default) cubic-bezier(0.25, 0.1, 0.25, 1.0) Speeds up then slows down
linear cubic-bezier(0.0, 0.0, 1.0, 1.0) Even, constant speed
ease-in cubic-bezier(0.42, 0, 1.0, 1.0) Starts slow and speeds up
ease-out cubic-bezier(0, 0, 0.58, 1.0) Starts fast and slows down
ease-in-out cubic-bezier(0.42, 0, 0.58, 1.0) Starts slow, speeds up, then slows down again
