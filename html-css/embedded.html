Embedding videos

Properly embedding videos in HTML pages involves several video formats and properties:

  • MP4
  • WebM
  • 3GP
  • MPEG
  • QuickTime
  • Ogg

Example:

<video controls>
     <source src="myVideo.webm" type="video/webm">
     <source src="myVideo.mp4" type="video/mp4">
     <p>Your browser doesn’t support HTML5 video. Here is a <a href="myVideo.mp4">link to the video</a> instead.
</video>

Attributes

Many of the video tag’s attributes are on/off switches; they are true when present and false when absent.

Property Value(s) Behavior
autoplay No value; this is boolean. Having it present in the video tag sets it to true. If set to true, the video automatically starts playing as soon as it can without stopping to buffer. Avoid setting to true unless necessary, since this is considered a bad user experience. Mozilla has a guide on proper use of the autoplay property.
controls No value; this is boolean. Having it present in the video tag sets it to true. If set to true, the video will have an overlay for controlling playback (play/pause, seek bar, volume).
height
width		 Pixel integer values with no unit, i.e. height="480" width="640". This sets the display area of the video.
loop No value; this is boolean. Having it present in the video tag sets it to true. If set to true, the video will automatically replay once it reaches the end.
muted No value; this is boolean. Having it present in the video tag sets it to true. If set to true, the video will begin playback with its audio muted.
playsinline No value; this is boolean. Having it present in the video tag sets it to true. If set to true, the video will play within its in-page container rather than fullscreen.
poster URL to an image file to be displayed while the video downloads. If set to true, the video will play within its in-page container rather than fullscreen.
preload none: The video should not be preloaded
metadata: Only the metadata (i.e. video length) is fetched
auto: The whole video should be preloaded even if the user is not expected to play it
[empty]: Defaults to auto		 This controls the preload behavior of a video. Note: autoplay has precedence over preload. If autoplay is specified, preload is also on by default.
src URL to the video file to embed. This is optional, and can only provide one source file. You can also use the <source> tag within the video element to specify one or more source files.

For the <source> tag inside the video element, it has three attributes:

  • src: URL to the path of the video file.
  • type: the format of the video file. Without this, the browser will download some of the video to see if it can play the file, which wastes time and bandwidth.
  • codecs: the codec used to encode the video, supplied within the type attribute.

Example: <source src="video/filename.mp4" type="video/mp4:codecs='avcl.42E01E, mp4a.40.2'" />
