Headings

Headings should always follow a logical hierarchy similar to an outline. Never skip past a heading level; use CSS to restyle headings if you need them to look different than their default heading level. (For example, the Sources heading at the bottom of some of these pages.)

Pages must only have one h1 heading for accessibility; this is the page title.

<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>

CSS font definitions

Shorthand

Font properties can be condensed into one shorthand definition. It must include a minimum of font-size and font-family. font-style, font-variant and font-weight must precede font-size. line-height must immediately follow font-size, preceded by / (i.e. 16px/3), and font-family must be the last value specified.

font: [font-style] [font-variant] [font-weight] [font-stretch] [font-size] / [line-height] [font-family];

font-family and font stacks

Fonts and typefaces should be defined in a stack with fail-overs in case fonts aren’t installed on the end user’s computer. Put font names made up of more than one word in double quotes.

Example:
body {
     font-family: Georgia, "Times New Roman", Times, serif;
}

font-size

Specify font size using one of the following units:

Unit Example Usage
Pixels 16px Gives precise control over the font size as defined by the height in pixels on the user’s screen.
Percentage 75% Sets the font size as a percentage relative to the default (inherited) font size. So with a default of 16px, 75% would be 12px.
Ems 1.5em Sets the font height using the inherited font size as 1em. For example: if the inherited font size is 12px, 1.5em would render at 18px.
Rems 1.5rem Sets the font height using the root font size of the page.

font-weight

This can be set to a numeric value (100–900 in increments of 100, normal/Roman is 400), or a text value (normal, bold, lighter, bolder).

font-style

Type Example Usage
normal Hello This is the standard display mode for text.
italic Hello Italicized text is used for emphasis or other visual cues; font designers normally include a variant of the font with special characters to look better when italic.
oblique Hello Oblique type is simply the normal type skewed a few degrees to resemble italic type. Fonts without specific italic characters will render as oblique.

Text transforms and decorations

Use text-transform to change text to UPPERCASE, lowercase, or Capitalize.

For text-decoration, the default parameter is none. You can change that to underline, overline, line-through, and other combined styles.

@font-face

You can load custom fonts into a stylesheet with @font-face, however, this downloads files to the user’s computer, and you need to make sure the font’s license permits you to do so. Generally, use web-safe fonts or Google fonts to avoid this problem.

Text indentation

Use text-indent and set the value to either a pixel amount or ems. This is similar to adding padding, but only applies to the first line of a paragraph.

Drop caps

Use the pseudo-classes :first-letter and :first-line to apply styles specific to those sections of a paragrah. The :first-line styling will re-render when the browser window changes size.

Columns

Use the column-count CSS property with an integer value to set the number of columns in text (Width auto-determined), and column-width to set the minimum width of columns and have the number auto-determined.

Example: column-count: 3; will have three auto-sized columns.
