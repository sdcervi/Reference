Components

Component Usage
<table> Containing element for a table
<tr> Table row
<td> Table cell (from ‘table data’)
<th> Table heading. Use scope="col" or scope="row" for better accessibility and readability
<thead> Container for column headings
<tbody> Container for data rows; put row-scoped th in their respective trs
<tfoot> Container for table footer, usually a sum or total row

Readability guidelines

  • Give cells padding: Give cell content enough padding so that it’s easy to read and doesn’t visually crash into the content of other cells.
  • Make headings look different: Style th headings bold or otherwise styled differently helps users read your tables more easily.
  • Shade alternate rows: Also known as zebra striping, this makes it easier to follow data along a row. Keep this subtle to avoid over-emphasizing shaded rows.
  • Align numbers right: Align numbers to the right to help distinguish between small and large values more easily, especially with monetary values. Use td { text-align: "." center; } to center-align numbers around the decimal point.
  • Manage the width: Don’t let tables get wider than their content supports. Tables with a lot of data work better at large sizes than small, simple tables. Limit table size if the data just won’t cover the whole screen at larger screen sizes. Wrap tables that will scroll off-screen on smaller screens in a figure element to let the browser handle scrolling.
