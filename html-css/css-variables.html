Why use CSS variables?

CSS variables allow you to store specific values in a reusable custom property.

One of the most intuitive use cases is for managing a color palette, or adjusting a stylesheet to be responsive. Rather than having to change every single instance of a color in a CSS document, variables allow you to change it just where the variable is defined and have it propagate through the document accordingly.

How to use CSS variables

Define variables using this syntax: --variablename. All variable names must begin with a double hyphen. Names are case sensitive; avoid using capital letters, to prevent confusion. Common practice is to hyphenate multiple words rather than camelCase them.

h1 {
     --main-header-color: #DADECC;
}

Use variables by referencing them:

h1 {
     --main-header-color: #DADECC;
     background-color: var(--main-background-color);
}

Variables can also be set to other variables. This allows you to create modular CSS rules and manage color schemes more effectively (for example, a dark mode vs light mode scheme).

html {
     --custom-purple: #FF64ED;
     --main-color: var(--custom-purple);
}

Fallback values

To prevent any errors with variables being invalid, you can provide fallback values as failsafes.

h1 {
     font-size: var(--main-header-font-size, 3rem);
}

The fallback value can also be a variable, at which point you can use nested var(...) function calls. This can be done as many times as needed, but avoid making fallback chains too long or you risk negatively impacting readability and maintainability.

h1 {
     font-size: var(--main-header-font-size, var(--fallback-header-font-size, 3rem));
}

Note: fallback variables will not fix browser compatibility issues, especially if a browser doesn’t support CSS variables.

Variable scope

Scope determines where a variable will work, or where value overrides are in effect. Variables can either be local or global scope.

Locally-scoped variables will only affect the specific HTML element that it is declared in, and any child elements. Parent elements and other unrelated elements won’t be able to access the variable’s value.

Globally-scoped variables are declared in the :root pseudo-class, and can be used across the entire HTML document. It’s common practice to define variables in the :root selecor, with any local overrides declared within their relevant selectors.

Since scoping limits how variables are defined and accessed, you can have multiple variables with the same name, in different scopes.

Inheritance

CSS variable values are inherited just like any other CSS property; if a variable is not explicitly redefined in a child element, it inherits the value from its parent element. Possible reasons to override an inherited value include redefining a color within one specific instance or element.

:root {
     --orange-color: #FF933A;
}

body {
     background-color: var(--orange-color);
} // References the color from root

button {
     --orange-color: #BF6317;
     color: var(--orange-color);
     border: 1px solid black;
} // Redefines the color and references its own redefinition
