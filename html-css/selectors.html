Selector types

Selector example Name Usage
* { ... } Universal selector Targets all elements on the page
type { ... } Type selector Targets elements by name
.class { ... }
type.class { ... }		 Class selector Targets all elements whose class attribute contains the specified one
#id { ... } ID selector Targets an element whose id attribute matches the specified one. IDs are unique and should only target one element. Use classes to target multiple elements.
parent>child { ... } Child selector Targets an element, class, or ID that is the direct child of another
parent child { ... } Descendant selector Targets all descendent elements within a parent, not just direct children
sibling1+sibling2 { ... } Adjacent sibling selector Targets the first sibling2 after any sibling1 elements
sibling1~sibling2 { ... } General sibling selector Targets a sibling2 that is a sibling of sibling1, even if it isn’t the first immediate sibling (i.e. any <p> following an <h2>, even if not the first one)

Specificity

Selectors of equal priority will always have the style defined further down in the stylesheet override the earlier one.

Styles

In order of increasing priority:

  1. External: <link rel="stylesheet" href="mystyle.css">
  2. Document head: <style>...</style>
  3. Inline: <tag style="padding: 1rem;">...</tag>

Selectors

In order of increasing priority:

  1. Type: type
  2. Class: .class, [attribute="value"], :pseudo-class
  3. ID: #id

Link pseudo elements

It’s recommended to define link pseudo-classes in this order:

  1. :link unvisited links
  2. :visited visited links
  3. :focus when a user tabs to a link using the keyboard
  4. :hover when a user places the mouse pointer over a link
  5. :active when a user clicks the link

Attribute Selectors

Format Usage Example
[attr] Represents elements with an attribute name of attr. a[href] { ... }
Any a element with an href attribute defined
[attr=value] Represents elements with an attribute name of attr whose value is exactly value. div[lang="en-us"] { ... }
All divs in US English
[attr~=value] Represents elements with an attribute name of attr whose value is a whitespace-separated list of words, one of which is exactly value. img[class~="logo"] { ... }
<img> elements whose class attribute contains the word "logo"
[attr|=value] Represents elements with an attribute name of attr whose value can be exactly value or can begin with value immediately followed by a hyphen, - (U+002D). It is often used for language subcode matches. div[class|="col"] { ... }
All div elements which have classes beginning with col
[attr^=value] Represents elements with an attribute name of attr whose value is prefixed (preceded) by value. a[href^="#"] { ... }
Internal links, beginning with "#"
[attr$=value] Represents elements with an attribute name of attr whose value is suffixed (followed) by value. a[href$=".org"] { ... }
Links that end in ".org"
[attr*=value] Represents elements with an attribute name of attr whose value contains at least one occurrence of value within the string. a[href*="example"] { ... }
Links with "example" anywhere in the URL
[attr operator value i] Adding an i (or I) before the closing bracket causes the value to be compared case-insensitively (for characters within the ASCII range). a[href*="insensitive" i] { ... }
Links with "insensitive" anywhere in the URL, regardless of capitalization
[attr operator value s] Adding an s (or S) before the closing bracket causes the value to be compared case-sensitively (for characters within the ASCII range). a[href*="cAsE" s] { ... }
Links with "cAsE" anywhere in the URL, with matching capitalization
