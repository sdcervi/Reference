CSS transitions

CSS transitions allow us to control the timing of visual state changes (i.e. a color change on mouseover). We can define:

  1. transition-property: Which CSS properties transition between two states’ defined properties
  2. transition-duration: How long a transition lasts, set in s or ms
  3. transition-property: The delay before a transition begins, set in s or ms
  4. transition-timing-function: The acceleration curve used in the transition

Shorthand:

transition: [transition-property] [transition-duration] [transition-timing-function] [transition-delay];

To combine multiple transitions into one definition, separate them with a comma:

transition: color 1s linear,
            font-size 750ms ease-in 100ms;

To use the same settings for all properties that transition on an element, use the keyword all in place of transition-property in a shorthand definition.

Animations can have accessibility concerns, especially with users with cognitive concerns, epilepsy, vestibular motion disorders, and other disabilities. Make sure that an interface doesn’t depend on solely on animation to convey meaning, since users can disable animation in their browsers in some cases.

Timing functions

These are shorthands for various cubic-bezier curves. The four coordinates specified in longhand curves (x1, y1, x2, y2) correspond with the x/y coordinates of the control handles for the origin and end point of the curve. Moving the handles by changing their coordinates changes the shape of the curve. Use this cubic bezier sandbox to experiment with different values.

Function Description
ease Equal to cubic-bezier(0.25, 0.1, 0.25, 1.0), the default value, increases in velocity towards the middle of the transition, slowing back down at the end.
linear Equal to cubic-bezier(0.0, 0.0, 1.0, 1.0), transitions at an even speed.
ease-in Equal to cubic-bezier(0.42, 0, 1.0, 1.0), starts off slowly, with the transition speed increasing until complete.
ease-out Equal to cubic-bezier(0, 0, 0.58, 1.0), starts transitioning quickly, slowing down the transition continues.
ease-in-out Equal to cubic-bezier(0.42, 0, 0.58, 1.0), starts transitioning slowly, speeds up, and then slows down again.
cubic-bezier(p1, p2, p3, p4) An author-defined cubic-Bezier curve
steps(n, [jumpterm]) Displays the transition along n stops along the transition, displaying each stop for equal lengths of time. Whether the transition holds temporarily at 0%, 20%, 40%, 60% and 80%, on the 20%, 40%, 60%, 80% and 100%, or makes 5 stops between the 0% and 100% along the transition, or makes 5 stops including the 0% and 100% marks (on the 0%, 25%, 50%, 75%, and 100%) depends on which of the following jump terms is used:
  • jump-start: Denotes a left-continuous function, so that the first jump happens when the transition begins
  • jump-end: Denotes a right-continuous function, so that the last jump happens when the animation ends
  • jump-none: There is no jump on either end. Instead, holding at both the 0% mark and the 100% mark, each for 1/n of the duration
  • jump-both: Includes pauses at both the 0% and 100% marks, effectively adding a step during the transition time
  • start: Same as jump-start
  • end: Same as jump-end
step-start Equal to steps(1, jump-start)
step-end Equal to steps(1, jump-end)

CSS transforms

CSS transforms change the shape and position of an element without disrupting the normal document flow around it. Transforms include rotation, skewing, scaling, and translation both in the x-y plane and in 3D space.

Note: only elements positioned by the box model can be transformed.

Two main properties define a CSS transform:

  • transform-origin: specifies the position of the origin. By default, it’s at the center of the element. This is most important when rotating, scaling, or skewing, since those need a specific point as a parameter.
  • transform: specifies the transforms to apply to the element as a space-separated list. They are applied in sequential order.
Transform Usage Example code Example render
rotate

Specifies rotation transforms. This allows for specification of a rotate independently of the transform property. Values can be in radians, degrees, or fractions of a full turn.

Note: The rotate transform is not fully supported by all major browsers. Use transform to specify rotations in these cases.

 rotate: 180deg;

This is rotated 180°
scale

Specifies how much to scale up or down an element. This allows for specification of a rotate independently of the transform property. Values greater than 1 make the element grow; 1 keeps it the same size; values less than 1 make it shrink.

Note: The scale transform is not fully supported by all major browsers. Use transform to specify scales in these cases.

 scale: 1.5;

This is scaled 1.5x
transform Allows you to rotate, scale, skew, or translate an element. Here’s the MDN documentation on it. transform: skewX(20deg);

This is skewed 20°
transform-origin Sets the origin around which a transformation is applied. For example, the center of rotation for rotate. Values are either position keywords (bottom, left, etc.) or distances, with up to three values corresponding to the X-, Y-, and Z-offsets. Empty values remain at their initial state. transform: rotate(10deg);
transform‑origin: top left;

This is rotated 10° around the top left corner
translate

Specifies translation transforms. This allows for specification of a translate independently of the transform property. Values can be in percentages or distances, with up to three values corresponding to the X-, Y-, and Z-offsets. Empty values remain at their initial state.

Note: The translate transform is not fully supported by all major browsers. Use transform to specify translations in these cases.

 translate: 10px 20px;

This is translated 10px right and 20px down
