Margin, padding & borders

HTML elements can be given margin, padding, and borders to style them and space them relative to elements around them.

How margins and borders interact with an element’s defined size is governed by which box-model setting you’ve applied. Click here for more information on the Box Model.

Margin collapse

Vertical (top and bottom) margins collapse between elements and only display the larger of the two defined margins.

Horizontal (left and right) margins do not collapse between elements, and will always add together to display the sum of the defined margins.

Borders

Borders, displayed between the margin and padding, can be defined in a variety of ways.

CSS shorthand: border: [border-width] [border-style] [border-color];. Note that this sets all four borders to be the same; shorthands for each parameter don’t work here.

For longhand, just target each side (i.e. border-top).

Rounded corners

To round the corners of buttons, images, or other HTML elements, use the border-radius CSS property. This can be set to any HTML-compatible unit: px, em, %, or rem.

When setting values individually, the order goes top-left, top-right, bottom-right, bottom-left.

To get a fully-rounded pill button, set the border radius to a measurement larger than 50% the smallest side. Setting it to 50% will instead give you an oval (or circle, if width and height are equal).
