Display

display changes how elements interact with each other, and enables you to change how elements flow around one another as well. The values are generally specified using two words, either hyphenated for legacy values, or two separate words for newer values.

Legacy values

Value Usage
block Block-level elements take up the full width of the page and don’t stack horizontally unless affected by position or float. This mode enables horizontal alignment at the element level, as well as manual height and width. Use display: block; or display: inline-block; to enable horizontal centering of images.
inline Inline elements include formatting tags such as em and strong. These can affect their contents, but aren’t able to have their own alignment properties, height, or width.
inline‑block This value allows elements to still display inline, while allowing you to set a manual height and width for the element.
flex This enables flexbox display properties on a block-level div. Read more about flexbox here.
inline‑flex This enables flexbox properties inside a div, and also allows divs with this display value to display inline with each other as well.
grid This enables CSS grid layout capabilities.
inline‑grid This enables CSS grid layout inside a div, and also allows divs with this display value to display inline with each other as well.
flow‑root Functions like display: block; and also establishes a new formatting context for its contents. This has the effect of clearing floats, and also affects z-index and transforms as well for any child elements.

Two-word syntax

New CSS display syntax uses two keywords. The first keyword defines the outside parameter, which is its role in the page flow layout (block versus inline). The second keyword defines the display context for the element’s contents.

CSS value Page flow display Contents display End result
block flow block flow Block display mode with standard element flow inside
inline flow inline flow Inline display mode with standard element flow inside
block flow‑root block flow-root Block display mode with a new formatting context (reset to defaults) for the contents
inline flow‑root inline flow-root Inline display mode with a new formatting context (reset to defaults) for the contents
block flex block flex Block display mode with contents set up as a flexbox
inline flex inline flex Inline-block display mode with contents set up as a flexbox
block grid block grid Block display mode with contents set up as a CSS grid
inline grid inline grid Inline-block display mode with contents set up as a CSS grid

Global values

Value Usage
none This removes the element from both the webpage and any accessibility outline. Don’t use this on content you want to be accessible to screen readers even when hidden visually.
inherit The element inherits its display parameters from the parent element
initial Sets the element’s display parameters to their default value

Other values

Value Usage
table The elements display as a block-level table
table-row The elements display like a standard <tr> element

Position

While display changes how elements flow relative to each other, position changes how they flow relative to the entire page. You can use its values to break an element out of the DOM without changing the code significantly.

Accessibility: Make sure that elements positioned with absolute or fixed do not obscure other content on the page when the page is zoomed to increase text size.

Value Usage
static The element is positioned with default settings. The top, right, bottom, left, and z-index properties will have no effect.
relative The element is positioned within the normal flow of the document, but can be shifted relative to itself. This doesn’t affect any other elements in the document. The space for the relative element is still the same as if it were set to static.
absolute The element is removed from the document flow, and no placeholder space is left for it. If an ancestor element has a position value, the child element will be positioned relative to that. If not, it will be positioned relative to the page. Margins of absolute positioned elements do not collapse like they do with static and other elements.
fixed The element is removed from the document flow, and no placeholder space is left for it. Its position will almost always be relative to the page, with a few rare exceptions. When printed, this element will be in the same position on every page of the printed document.
sticky The element is positioned within the normal flow of the document, but when scrolled past, “sticks” to the top of the nearest scrolling ancestor (usually the viewport). At least one offset value (top, right, bottom, left) must be set or this will function like relative positioning.

Z-index

Overlapping elements can be reordered visually by using z-index. The higher the number, the closer that element’s layer is to the “front” of the page. Values must be an integer, and best practice is to carefully manage your layer numbers to avoid lazy fixes like 9999 to force elements to the top. This works for simpler webpages, but as they get more complex and need more elements’ layer position defined, it can cause conflicts and unpredictable rendering.

Offset

Use the top, right, bottom, and left properties to change an element’s position. Values can be either 0, defined in ems, percentages, or pixels.

Float

A float property moves an element as far left or right in a container as it can possibly go. The element is removed from the normal flow of the page, but other content still wraps around it.

Value Usage
left The element is floated to the left side of its containing block.
right The element is floated to the right side of its containing block.
none The element does not float and flows according to its default properties.
inline-start The element floats on the starting side of its containing block; this is on the left for left-to-right languages, and on the right for right-to-left languages.
inline-end The element floats on the ending side of its containing block; this is on the right for left-to-right languages, and on the left for right-to-left languages.

Clearing floats

To force elements under a floated element, use the clear property:

Value Usage
none The element is not moved down past any floated elements.
left The element is moved down past only left-floated elements.
right The element is moved down past only right-floated elements.
both The element is moved down past any previous floated elements. This is most useful when using an empty div to clear floats between page sections.
